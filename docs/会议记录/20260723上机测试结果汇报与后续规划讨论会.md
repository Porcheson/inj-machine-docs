## 二、核心要点

### 2.1 鲁班平台架构与近期进展

- **分层架构**：平台采用分层架构，将上层工艺、中间层基础动作与底层动力系统进行解耦，以实现各层间的独立开发和维护。
- **近期功能开发**：已完成多项功能开发，包括：
  - 为定位功能块增加压力线幅
  - 为速度功能块增加压力线幅
  - 引入多级队列缓存
  - 引入加速度平滑等功能

### 2.2 合模过程中的问题分析（上机测试结果）

#### 速度与压力反馈

通过分析采样曲线发现，合模与开模过程中的速度反馈与目标曲线吻合度尚可，但在模式切换（如位置模式到压力模式）时，会出现速度骤降甚至为零的现象，不符合预期。

#### 系统性问题

| 问题类别 | 现象描述 |
| -------- | -------- |
| 速度超调与振荡 | 在射胶保压等阶段，系统会出现超调和振荡现象 |
| 压力波动 | 在高压阶段，压力值存在轻微的波动 |
| 参数优化 | 当前的控制算法在处理高速高压和低压时，其性能表现尚不理想 |

### 2.3 后续工作与优化方向

#### 核心优化

优先解决合模过程中动模板速度与油缸速度不匹配的问题。计划通过建立曲轴连杆机构的运动学模型，采用查表法来实现更精确的速度映射。

#### 参数规划

讨论了当前参数规划方式（如斜率、预调等）占用大量地址的问题，提出了未来新平台应进行优化，以提升资源利用率。

#### 测试与验证

明确了后续将分阶段进行测试，先解决当前版本存在的问题，待系统趋于稳定后再进行精度和重复性的全面测试。
